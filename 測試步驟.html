🧪 TQM 系統本地測試完整步驟

ℹ️ 測試目的：在本地環境測試所有功能，確認系統可正常運作後再遷移到公司伺服器

📋 測試前準備檢查表

  • 已登入 TQM 系統 Firebase 模式
  • 確認當前資料完整（可先瀏覽各功能確認）
  • 準備好接收測試結果的記錄文件
  • 確保網路連線穩定

🎯 第一階段：資料備份

1 開啟系統設定頁面
1. 點擊左側選單的「系統設定」
2. 捲動到頂部「資料庫模式設定」區塊
3. 確認當前模式顯示為「Firebase 雲端資料庫」
2 匯出 Firebase 完整備份
1. 點擊「匯出 Firebase 資料」按鈕
2. 等待匯出完成（檢查 Console 顯示匯出進度）
3. 下載的檔案名稱類似：TQM_Firebase_備份_2026-1-13.json
4. 檢查檔案大小（應該在 4-5 MB 左右）
5. 將檔案保存到安全位置（建議複製多份備份）
✅ 檢查點：開啟下載的 JSON 檔案，確認包含以下內容：
  • version: "4.0"
  • tqm_users, rd_projects, rd_tasks 等集合資料
  • localStorage 資料
  • stats 統計資訊

🔄 第二階段：匯入本地資料庫

3 執行資料匯入
1. 在「資料庫模式設定」區塊，點擊「匯入到本地資料庫」
2. 確認彈出的對話框，點擊「確定」
3. 等待匯入完成（約 10-30 秒，取決於資料量）
4. 查看成功訊息：「✅ 資料匯入成功! 請切換到本地模式測試。」

開啟瀏覽器 Console 檢查匯入詳情：
- 按 F12 開啟開發者工具
- 查看 Console 分頁
- 應該顯示類似以下訊息：
📥 開始匯入 Firebase 備份資料...
📦 匯入 tqm_users: 20 筆
📦 匯入 rd_projects: 224 筆
...
✅ 匯入完成! 總計 411 筆資料，涵蓋 19 個集合
💡 技術說明：資料已匯入瀏覽器的 IndexedDB，可在開發者工具 → Application → IndexedDB → TQM_LocalDB 查看

🔀 第三階段：切換到本地模式

4 切換資料庫模式
1. 點擊「本地模式」按鈕
2. 確認彈出的對話框「切換到本地模式將重新載入頁面，確定要切換嗎？」
3. 點擊「確定」
4. 頁面自動重新載入
5. 重新載入後，檢查「資料庫模式設定」區塊
6. 確認顯示「本地資料庫 (IndexedDB)」
✅ 檢查點：系統現在使用本地 IndexedDB，不再連線到 Firebase

4. 新增任務到專案

⚠️ 注意：
  • 本地模式的資料修改不會同步到 Firebase
  • 測試時的所有變更只影響本地資料
  • 若要恢復原始資料，重新匯入即可

📊 第五階段：效能驗證

6 檢查系統效能
開啟瀏覽器 Console（F12），查看以下指標：

1. 資料載入速度
- 查看訂閱成功訊息的時間戳記
- 本地模式應該比 Firebase 更快

2. 操作回應時間
- 新增/修改/刪除操作應該立即反應
- 無需等待網路延遲

3. 記憶體使用
- 開發者工具 → Performance → Memory
- 確認無記憶體洩漏

🔄 第六階段：切回 Firebase（可選）

7 恢復 Firebase 模式
測試完成後，如需切回 Firebase：

1. 進入「系統設定」
2. 點擊「Firebase 模式」按鈕
3. 確認切換
4. 頁面重新載入
5. 系統恢復使用 Firebase 雲端資料

注意：本地測試期間的資料變更不會帶回 Firebase

8 完成測試報告
建議記錄以下資訊：

❌ 發現的問題：

- 記錄重現步驟

- 操作回應時間

🚀 後續遷移計畫

📋 確認本地測試無誤後的遷移步驟：
  1. 從 Firebase 匯出最新完整資料（包含測試期間的更新）
  2. 在公司伺服器建立新的資料庫
  3. 將備份資料匯入新資料庫
  4. 修改系統程式碼，連線到新資料庫
  5. 部署到公司伺服器
  6. 進行最終測試
  7. 正式上線

❓ 常見問題

Q: 本地資料會佔用多少空間？

A: 根據目前資料量，約 5-10 MB，IndexedDB 預設上限為 50 MB（部分瀏覽器更大）。

Q: 清除瀏覽器資料會刪除本地資料庫嗎？

A: 是的，清除瀏覽器資料會刪除 IndexedDB。建議定期匯出備份。

Q: 可以在不同電腦使用本地模式嗎？

A: 每台電腦的 IndexedDB 是獨立的，需要分別匯入資料。

Q: 本地模式可以多人使用嗎？

A: 不建議。本地模式適合單人測試，正式環境應使用 Firebase 或公司伺服器。

Q: 如何確認目前使用的模式？

A: 查看「系統設定」頁面的「資料庫模式設定」區塊，或開啟 Console 查看初始化訊息。
